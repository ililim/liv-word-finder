# Liv Word Finder

A mobile-first word-finding web app, built as a gift for a friend (Liv). It assists with
word games — Scrabble and the NYT games (Wordle, Spelling Bee, Letter Boxed, crosswords) —
by finding words from available letters and/or letter-position constraints.

## ⚠️ Keep this file updated

**Update this CLAUDE.md continuously as the project evolves** — new decisions, word list
changes, chosen designs, architecture, deployment steps, gotchas. Treat it as the living
source of truth for the project. This instruction itself came from the project owner and
should never be removed.

## Product constraints

- **Mobile-first, iPhone specifically.** It will be used almost exclusively on a phone.
  Design for one-handed thumb use, ~390px viewport, large touch targets.
- **It's a website, not a native app.** The friend will "Add to Home Screen" in Safari, so
  it should behave like an app: PWA-style meta tags (`apple-mobile-web-app-capable`,
  `viewport-fit=cover`, theme color, app icons), no reliance on browser chrome.
- **Client-side only.** Plain JavaScript web app; word lists shipped/loaded client-side.
  No backend needed. Likely published via GitHub Pages eventually (currently local only,
  not yet a git repo).
- **Multiple selectable word lists** (dictionaries), because different games use different
  lexicons.

Notes:
- **NYT Spelling Bee has no public dictionary.** NYT curates it by hand; it's *smaller*
  than Scrabble lexicons (no obscure words, minimum word length 4, and the letter S never
  appears in the puzzle). Best approach: filter ENABLE or NWL for length ≥ 4 and ≤ 7
  unique letters, and accept some false positives. Community solvers all do this.
- **Licensing:** NWL and CSW are proprietary (NASPA / Collins-HarperCollins). Fine for a
  private gift app; don't advertise redistribution if published publicly. ENABLE is public
  domain — the safe default list.
- Scrabble lexicon files with definitions are useful for a "check word / show definition"
  feature. CSW21.txt line 1 is a `#` license comment.

## Design direction

### Round 1 (`docs/v1_explorations/`, 20 mockups)

Explored two axes: **tools** (anagram solver, pattern matcher, Wordle helper, Spelling Bee
honeycomb, Letter Boxed solver, score maximizer, hook explorer, word checker, cheat
sheets) and **layouts** (bottom tab bar, game-picker home, smart single search box,
swipeable deck, chat assistant, widget dashboard, minimal single-purpose). Gallery:
`docs/v1_explorations/index.html` (flat 01–20 grid, per owner preference).

**Owner feedback on round 1 (2026-07-13):**
- Favorites: **08 Pattern Slots, 14 Thumb Board, 16 Live Split** (design + functionality).
- **Visual style winner: 14** (bold, high-contrast, heavy uppercase type, loud accent) —
  BUT its keyboard keys are way too big; use compact ~44–48px keys like 08.
- Likes the idea of a bottom nav (tools + settings) for the real app; within tools, a
  bottom mode bar toggling input modes (slots ↔ free-typed pattern etc.).
- Wants "add one letter / subtract one letter" tools (a feature from the recipient's own
  homemade app) where reasonable.

### Key product philosophy (from owner, round 2)

**Assist, don't solve.** The recipient (Liv) plays Scrabble & NYT puzzles against bots
when tired. She wants help finding words but NOT the solution handed to her — she wants
to feel like she recalled the word herself. So: no "best play" rankings, no full Letter
Boxed solutions. Prefer possibilities, counts, filters, and gradual/optional hints.

**Her current homemade app** ("Word Lookup", plain desktop web): dictionary lookup,
letter solver, letter filter, word pattern w/ match highlighting, "add one letter"
button, "exclude double letters" toggle. Don't copy its design (very basic) — but these
features are her actual daily toolkit; keep them all covered.

### Round 2 (`docs/v2_explorations/`, 10 mockups)

One unified visual family: 14's style DNA + 08's compact keys, chartreuse accent, heavy
uppercase display type, pill segmented controls, live counters, light+dark. Concepts:
01 Mode Board (flagship: bottom mode bar — Slots/Pattern/Rack/±1) · 02 Word Lab ±1 ·
03 Letter Sets (allow/require/exclude keys, no-doubles toggle) · 04 Parallel (stacked
searches with chain-link) · 05 Anchors (play-through finder) · 06 Nudge (masked results,
escalating reveals) · 07 Chunks (combo include/exclude) · 08 Recall First (count only,
she guesses, then reveal missed) · 09 Trace (Letter Boxed square as rule-enforcing input,
count-only hints) · 10 Live Pro (16 evolved). Gallery: `docs/v2_explorations/index.html`.

## The app (v0.1, 2026-07-13)

Three tools behind a top pill switcher. **Each mode owns a color** (owner direction):
SLOTS red · PATTERN chartreuse · LADDER amber — the active nav segment, counters, and
accent states wear the mode's color. **Hard corners everywhere** (`--r: 0`); rounded
shapes survive only in the nav (seg pill + header ?/⚙ buttons). Header: brand left,
"?" (combined help sheet) and "⚙" (dictionary) right.
- **SLOTS** — exact-length positional search. Native keyboard types through an invisible
  "ghost" input (no custom typing keyboard); space = any; slots shrink to fit up to 12
  per row (smaller font when narrow), then wrap — never overflow. Below: a QWERTY
  **constraint board** — tap cycles blank → MAY (inverted) → MUST (accent + dot); once
  anything is tapped, untapped letters are NOT allowed (they dim). Slot letters are
  implicitly allowed.
- **PATTERN** — free-typed; space→`_` (any letter), `*` = any run; START/END WITH anchor
  buttons flank the input (both off = contains); length chips with ALL (= no filter,
  on by default; tapping it clears the others); "?" opens a help sheet (placeholder
  example: B_TT*Y → BATTERY · BUTTERY · BITTERLY).
- **LADDER** (±1, internal id "lab") — one letter more/less; tap a word to walk to it
  (trail crumbs grow, ✕ CLEAR button at trail end; **typing directly resets the trail**);
  hold a word for its definition. Toggles: ANAGRAM (rearranged letters, on by default),
  UNIQUE (no repeated letters), +S (hide S-plurals). +1/−1 sections split on a thick accent rule with signed badges. No "word not
  in dictionary" notices (owner cut it).
- Constraint board MUST keys show an inline ✱ superscript (required-field convention).
- Filter rows: ↺ reset sits LEFT, terse filter chips RIGHT, one line, no inline hint text
  (the ? help sheet teaches instead). No inline ✕ in inputs (owner dislikes).
- Everywhere else: tap a word → definition sheet (word, NWL/CSW/ENABLE flags, definition).
- Dictionary picker in header: NWL2023 (default) / CSW21 / ENABLE / ALL (union).
- Global rack drawer is milestone 2 — `state.rack` + engine `fitsRack` already wired.

**Product rules:** never display points/scores (owner: keeps it fun). Assist, don't solve.
Explain unintuitive things with one short hint line, no verbosity — she learns fast.

**Design rules:** radius system: `--r: 8px` for every rectangle, `--r-pill` only for
standalone pills (never nest pills in rects); concentric nesting where nested radii are
unavoidable (seg: outer 21 / inner 17 with 4px padding). Owner flagged mismatched nested
radii as messy — keep the scale tight.

**Code style:** modeled on `~/dev/agents/kino` (owner's reference for beautiful code):
`// — Section ————` dividers, terse single-line methods, comments explain *why* only,
guard clauses/early returns, domain-first naming, `///` doc headers. Keep that voice.

## Anti-slop design principles (round 3 lesson, 2026-07-13)

The first 10 v3 skins read as "blurry glossy AI slop" (owner). Researched why AI design
fails (smoothui.dev/blog/ai-design-slop, impeccable.style/slop, 925studios.co ai-slop
tells, prg.sh purple-gradient piece). Root cause: models regress to the statistical
median of training data — glow, blur, gradients, timid palettes, sameness. **Rules for
all future visual work on this project:**

1. **Crisp over glossy.** No backdrop blur, no glow shadows, no gradients (unless the
   gradient IS the concept, with hard stops). Shadows: none, or hard offset.
2. **Readable always.** Ink-on-paper contrast first; never color-on-color text below AA.
3. **One point of view per design, from a real tradition** (Swiss modernism, editorial
   print, transit signage, risograph, split-flap boards...) — never "modern app median."
4. **Semantic color, few colors.** Every hue has a job; no decorative color.
5. **Hierarchy via type scale/weight contrast**, not via color or boxes. Deliberate font
   choices (system stacks are fine; the *choice* must be deliberate).
6. **No slop tells:** no thick colored border-on-one-side cards, no 24px+ blob radii, no
   icon-tile-over-heading, no hairline-border-plus-soft-wide-shadow, no neon cyan/magenta
   on navy, no purple.
7. Rhythm: tight groupings, generous section separation; avoid uniform card-grid monotony.
